# Simon: 基于 LabVIEW 开发的定向迷宫记忆游戏

> 💡 **项目简介**
> 本项目是对经典记忆力游戏“Simon Says”的一次颠覆性重构。本作脱离了传统的“按顺序点击色块”模式，采用**“定向迷宫”**的设计理念。玩家需要操纵角色，穿梭于一个个未知的房间，依靠记忆重现系统给出的迷宫路线，最终到达终点。本项目 100% 使用 LabVIEW 开发，展示了图形化编程在复杂游戏逻辑、UI 视窗跟随以及模块化架构设计上的强大能力。

## 🎮 创新玩法 (Innovative Gameplay)

* **空间路线记忆**：游戏开始时会展示一条通往终点的路径，玩家需要记住这个移动序列（如：上、右、右、下...），随后凭借记忆操纵角色穿越迷宫。
* **视窗跟随与局部视野**：UI 呈现为以玩家为中心的跟随窗口。整个虚拟迷宫在后台无缝滑动，而前面板**只展示玩家当前所在的房间**。玩家需要极强的空间方向感来判断自己在全局迷宫中的位置。
* **动态门禁反馈逻辑**：房间的 4 个方向门通过开/关状态实时反馈玩家的操作对错。
    * **路线正确（On-path）**：当前房间的所有门全部打开，迷惑性拉满。
    * **路线错误（Off-path）**：当玩家偏离规定路线时，系统会进行限制，此时**只有返回上一个房间的门会打开**，强制玩家退回并修正路线。
* [cite_start]**多玩家与密码控制系统**：内置了完整的登录面板，支持多玩家账户管理与密码验证控制 [cite: 32, 156]。

## 🛠️ 核心架构设计 (Architecture & Modules)

本项目的底层代码放弃了简单的面条式连线，采用了高度解耦的**三模块架构 (Three-Module Architecture)**，各模块通过定义良好的 API 和事件广播进行通信：

### 1. Main 模块 (UI 与主控)
负责前面板的展示与顶层交互响应：
* [cite_start]**事件处理**：拦截并处理键盘输入（Idle 状态）以控制角色移动 [cite: 30]。
* [cite_start]**UI 渲染**：动态更新当前房间的画面展示（UI: Update），以及根据底层数据刷新角色的位置和门的开关状态（API: Update Position / Update Gate） [cite: 31, 50, 54]。
* [cite_start]**流程控制**：管理游戏的宏观生命周期，包括启动登录面板、新玩家初始化（Action: Launch Game），以及打开菜单并暂停底层计时器（Action: Open menu） [cite: 32, 33]。

### 2. Observer 模块 (运动与碰撞监测)
作为“物理引擎”的核心，实时监控玩家的移动意图：
* [cite_start]**方向响应**：处理上下左右的移动指令（API: Move Up / Down / Left / Right） [cite: 84, 90, 96, 102]。
* [cite_start]**环境交互**：在移动后观察并判定结果（是否发生墙壁碰撞？是否成功进入了某扇门？），并向下游广播房间切换（Switch Room）事件 [cite: 85, 91, 101]。

### 3. Scheduler 模块 (规则调度与计时引擎)
游戏的“大脑”，负责所有暗部逻辑的推演：
* [cite_start]**关卡与路径生成**：根据当前难度等级生成正确的迷宫路径，并重置玩家的路径点（API: Begin） [cite: 124, 125]。
* [cite_start]**路径校验与门禁调度**：实时比对玩家走入的门与预设的 Waypoint 是否一致。根据校验结果，动态决定下一个房间门的开关状态（API: Check Waypoint），并扣除失败次数 [cite: 131, 132]。
* [cite_start]**全局计时器**：在后台独立运行（Idle 状态下持续追踪），并提供启动、暂停、根据暂停时长更新结束时间等功能，一旦时间耗尽则强制结束当前回合 [cite: 116, 140, 145, 149]。
